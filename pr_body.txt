## Summary

Remove all CLI arguments - all configuration now in settings.py.

## Changes

### build_database.py
- Remove `argparse` import and all CLI argument parsing
- Remove `--resume` and `--async` flags
- Update `build_database()` signature to remove parameters
- All runtime options now read from `settings.py`

### settings.py
- Add `resume: bool = False` - resume from checkpoint
- Add `use_async: bool = False` - use async CDX client (10x faster)

### README.md
- Update documentation to reflect new configuration approach

## Before

```bash
# CLI arguments
uv run build_database.py --resume --async
```

## After

```python
# In settings.py
from settings import settings

settings.resume = True  # Resume from checkpoint
settings.use_async = True  # Use async CDX client (10x faster)
```

```bash
# No arguments needed
uv run build_database.py
```

## Rationale

1. **Cleaner code** - No CLI parsing overhead
2. **Single source of truth** - All configuration in one place
3. **Version control** - Easy to commit different configurations
4. **User preference** - Matches preferred style (constants over CLI args)

## Migration

If you were using CLI arguments, update your settings.py:

```python
# Old way (CLI)
uv run build_database.py --resume --async

# New way (settings.py)
from settings import settings
settings.resume = True
settings.use_async = True
uv run build_database.py
```
