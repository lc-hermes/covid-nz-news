## Summary

Major production improvements for COVID NZ News database builder.

## New Features

### 1. Progress Checkpointing (progress.py)
- Save progress after each crawl-domain pair
- --resume flag to continue from checkpoint
- Prevents losing progress on interruption
- 12 new unit tests

### 2. Publish Date Extraction
- Extract article publish date from HTML metadata
- Supports: article:published_time, datePublished, og:article:published_time, etc.
- Added publish_date column to database schema
- Index on publish_date for faster queries

## Changes

### New Files
- progress.py - Progress tracking module
- tests/test_progress.py - Unit tests (12 tests)

### Modified Files
- build_database.py - Added --resume flag and progress integration
- database.py - Added publish_date column and index
- settings.py - Added checkpoint_file config

## Testing
- 66 tests passing (12 new tests added)
- All lint and type checks passing

## Usage

```bash
# Start fresh build
uv run build_database.py

# Resume from checkpoint after interruption
uv run build_database.py --resume
```

## Benefits

1. **Reliability**: No more losing hours of progress on interruption
2. **Flexibility**: Can pause and resume at any time
3. **Better Data**: Publish dates enable accurate timeline analysis
4. **Production Ready**: Suitable for long-running builds
